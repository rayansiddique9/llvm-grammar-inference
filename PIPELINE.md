# Complete PolyTracker → MIMID Grammar Inference Pipeline

**Purpose:** Step-by-step guide to generate grammar from parser using PolyTracker instrumentation

**Environment:** macOS with Docker (also works on Linux natively)

---

## 📋 **PREREQUISITES**

### **Required Software:**
```bash
# Docker (for PolyTracker)
docker pull trailofbits/polytracker:latest

# Python 3.x with packages
pip install polytracker cxxfilt pudb
```

### **Required Files:**
- Parser source code (e.g., `tiny.c`)
- One or more test input files
- MIMID grammar inference scripts (in `src/grammar_inference/`)

## 🚀 **COMPLETE PIPELINE (11 STEPS)**

### **PHASE 1: PolyTracker Instrumentation (Steps 1-3)**

#### **Step 1: Compile Parser with PolyTracker**

Two sub-steps required — `instrument-targets` needs a blight journal generated by `build` first.

```bash
cd examples

# Step 1a: Build with blight (generates blight_journal.jsonl + bitcode files)
docker run --rm --platform linux/amd64 \
  -v "$(pwd):/workdir" -w /workdir \
  trailofbits/polytracker \
  polytracker build clang -o tiny tiny.c

# Step 1b: Instrument (reads blight_journal.jsonl, generates tiny.instrumented + functionid.json)
docker run --rm --platform linux/amd64 \
  -v "$(pwd):/workdir" -w /workdir \
  trailofbits/polytracker \
  polytracker instrument-targets --taint --ftrace --cflog tiny

# Or use the provided script:
bash instrument.sh
```

**Output:**
- `tiny.instrumented` - **Instrumented binary** (don't commit to git!)
- `functionid.json` - Function name mappings (small, can commit)
- `blight_journal.jsonl`, `tiny.bc`, `tiny.opt.bc`, `tiny.instrumented.bc` - Intermediate (gitignored)

**Verify:**
```bash
ls -lh tiny.instrumented functionid.json
file tiny.instrumented
# Should show: ELF 64-bit LSB executable, x86-64, dynamically linked
```

**⚠️ Note:** The `.instrumented` binary is regenerable and platform-specific (Linux). Add `*.instrumented` to `.gitignore`.

#### **Step 3: Run Instrumented Parser (Per Input)**

```bash

export POLYTRACKER_STDIN_SOURCE=1
export POLYTRACKER_LOG_CONTROL_FLOW=1

# Run instrumented binary on ONE input
docker run --rm --platform linux/amd64 \
    -v $(pwd):/workdir -w /workdir \
```

**Output:**
- `polytracker_cf.tdag` - Control flow trace (can be VERY large - 33GB for 6 bytes!)

### **PHASE 2: Extract MIMID Format (Steps 4-5)**

#### **Step 4: Create Docker Wrapper (One-Time Setup)**

```bash
# Create test_subject.sh wrapper
cat > test_subject.sh << 'EOF'
#!/bin/bash
# Wrapper for instrumented binary to work with grammar-miner validation

if [ $# -eq 0 ]; then
    echo "Usage: $0 <input_file>" >&2
    exit 1
fi

INPUT_FILE="$1"

if [ ! -f "$INPUT_FILE" ]; then
    echo "Error: Input file '$INPUT_FILE' not found" >&2
    exit 1
fi

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

# Run instrumented binary inside Docker
docker run --rm --platform linux/amd64 \
    -v "$SCRIPT_DIR:/workdir" \
    -w /workdir \
    trailofbits/polytracker \
    bash -c "./tiny.instrumented < '$INPUT_FILE'" 2>/dev/null

exit $?
EOF

chmod +x test_subject.sh

# Test wrapper
./test_subject.sh tiny.input.1
# Should run without errors
```

## 🐛 **TROUBLESHOOTING**

### **Issue: "cannot execute binary file"**

**Symptom:**
```
./tiny.instrumented: cannot execute binary file
```

**Cause:** Trying to run Linux binary natively on macOS

**Fix:** Always run via Docker:
```bash
docker run --rm --platform linux/amd64 \
    -v $(pwd):/workdir -w /workdir \
    trailofbits/polytracker \
    bash -c "./tiny.instrumented < input"
```

---

### **Issue: "No such file or directory: 'polytracker_cflog'"**

**Symptom:**
```
FileNotFoundError: [Errno 2] No such file or directory: 'polytracker_cflog'
```

**Cause:** "original" field not updated to executable wrapper

**Fix:**
```bash
python3 fix_original_field.py
# Then re-run grammar-miner.py
```

---

### **Issue: parse_trees.json has "<>" instead of "<START>"**

**Symptom:**
```json
"tree": ["<>", ...]
```

**Cause:** Method 0 has empty string `""` instead of `null`

**Fix:** Check extract_polytracker_to_mimid.py:
```python
# Should be:
new_method_map["0"] = [0, None, [1]]  # ✓ None → null → <START>

# NOT:
new_method_map["0"] = [0, "", [1]]    # ✗ "" → <> (wrong!)
```

---

### **Issue: "assert node_name == '<START>'" fails**

**Symptom:**
```
AssertionError at generalizemethod.py:156
assert node_name == '<START>'
```

**Cause:** Parse tree root is not `<START>`

**Fix:** Check parse_trees.json:
```bash
head -5 parse_trees.json
# Must show: "<START>"
```

If not, re-run token extraction with correct synthetic root.

---

### **Issue: Grammar is too flat / missing structure**

**Symptom:** Professor says "grammar has gone flat"

**Cause:** 
1. Only 1 input file used
2. grammar-compact.py removed too much

**Fix:**
1. Use `parsing_grammar.json` instead of `final_grammar.json`
2. Add more input files (3-5 recommended)
3. Re-run entire pipeline with multiple inputs
